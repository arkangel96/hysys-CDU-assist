# Lessons (curated PE notes)

**Purpose:** Capture what we learned from Assist trials — mistakes and keeps — so the next feed/session does not repeat blind errors.

**Not:** an auto-learning database that changes `propose_action`.  
**Full learning/memory engine:** still **HELD** in Inventory (`new_intelligence` Deliverable 4 / expert module 36).

## How to use

1. After a useful Trial Map step (KEEP, REVERSE, or clear mistake), copy [`_TEMPLATE.md`](_TEMPLATE.md).
2. Name: `YYYY-MM-DD_<column>_<short_topic>.md` (example: `2026-07-23_T-100_rr_wrong_active.md`).
3. Fill **Feed/assay**, **Column tag**, and mark each takeaway **Reusable** vs **Case-specific**.
4. Link the Trial Map / `docs/trials/` note if one exists.
5. Do **not** paste magic GoalValues into global CDU rules — only reusable symptom → family / do-not patterns.

## Scope (multi-feed)

- **Column tag** (e.g. T-100) = HYSYS equipment name for this case — not “forever plant.”
- **Feed / assay** will change; duties, rates, T, P will move.
- Lessons must survive feed change by stating **patterns**, not tonight’s USGPM / Btu/hr as eternal truth.
