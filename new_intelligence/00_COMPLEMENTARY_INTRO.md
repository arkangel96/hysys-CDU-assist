# Engineering Intelligence Framework — Complementary Intro

## Purpose

This folder (`new_intelligence/`) holds a **complementary** engineering intelligence package for Aspen HYSYS **simple distillation** assistance.

It is meant to **help and strengthen** the existing Simple Column Assist v1 New Intelligence — not replace it, not override it, and not fight it.

---

## Relationship to existing intelligence (non-negotiable)

| Source | Role |
|--------|------|
| Existing docs + Inventory v1 + coded Assist | **Active judgment today** (States, FINAL_TARGET, trials, COM safety) |
| This package (Deliverables 1–6) | **Complementary OS** — mindset, knowledge, interaction, learning, rules, workspace |
| Both together | Same PE direction; clarify and extend — **no rival brain** |

### Rules

- Do **not** treat this package as superseding older engineering docs.
- Do **not** declare older rules inactive.
- Do **not** combine conflicting advice as two competing authorities.
- When two texts differ, **reconcile them** so they agree; until reconciled, prefer the **more specific validated Assist rule** (especially FINAL_TARGET lock, DOF, no auto-save, one MV).
- New ideas from this package become active only after they are reviewed and fitted next to Inventory v1 — as **additions**, not replacements.

---

## Engineering priority (complementary use)

Use sources together in this helpful order (guidance, not a war of supremacy):

1. Validated Assist / Inventory rules for this product  
2. This framework’s reasoning + knowledge + HYSYS interaction specs  
3. Project-specific notes  
4. Historical lessons (reference)  
5. General PE knowledge  

If something conflicts, **fix the conflict in discussion** — do not let the AI silently pick a winner that relaxes plant targets or skips state diagnosis.

---

## Engineering identity

Together, these materials teach the AI to think like a senior process engineer on **simple distillation columns** in HYSYS:

- Observe and validate before changing  
- Diagnose before optimizing  
- One bounded experiment at a time  
- Preserve a rollback path  
- Explain every recommendation  

This package is part of the engineering operating support — **alongside** Simple Column Assist v1, not instead of it.
